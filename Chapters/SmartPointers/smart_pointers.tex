\section{Smart Pointers in C++}
\href{https://en.cppreference.com/w/cpp/memory}{Cppreference documentation about \textbf{smart pointers}}. 
\newline
\textbf{Smart pointers} enable automatic, exception-safe, object 
lifetime management.
\newline
Generally speaking, a \textbf{smart pointer} is a data type whose value 
can be used like a pointer, but which provides the additional feature of 
memory management which frees the memory when the smart pointer is 
no longer used. A smart pointer should therefore be used any time 
the ownership of memory needs to be tracked.

%% Unique Pointer
\subsection{\texttt{std::unique\char`_ptr}}
    \href{https://en.cppreference.com/w/cpp/memory/unique_ptr}{cppreference documentation about \textbf{\texttt{std::unique\char`_ptr}}}.
    \newline
    \texttt{std::unique\char`_ptr} is a smart pointer that \textbf{owns and manages} 
    another object through a pointer and \textbf{disposes of} that object 
    when the \texttt{std::unique\char`_ptr} goes out of scope.
    \newline
    The object is disposed of, using the associated deleter when either 
    of the following happens:
    \begin{itemize}
        \item the managing \texttt{std::unique\char`_ptr} object is destroyed
        \item the managing \texttt{std::unique\char`_ptr} object is assigned 
        another pointer
        \newline 
        via \texttt{operator=} or \texttt{reset()}. 
    \end{itemize}

\toclessnparagraph{Notes}
Only \textbf{non-const} \texttt{std::unique\char`_ptr} can 
\textbf{transfer the ownership} of the managed object to another 
\texttt{std::unique\char`_ptr}.
\newline
If an object's lifetime is managed by a const \texttt{std::unique\char`_ptr}, 
it is limited to the scope in which the pointer was created.

\texttt{std::unique\char`_ptr} is commonly used to manage the lifetime 
of objects, including:
    \begin{itemize}
        \item providing \textbf{exception safety} to classes and functions 
        that handle objects with dynamic lifetime, 
        by \textbf{guaranteeing deletion} on both normal exit and 
        exit through exception 

        \item \textbf{passing ownership} of uniquely-owned objects with 
        dynamic lifetime into functions 

        \item \textbf{acquiring ownership} of uniquely-owned objects 
        with dynamic lifetime from functions 

        \item as the element type in move-aware containers, 
        such as \texttt{std::vector}, which hold pointers to 
        dynamically-allocated objects
        (e.g. if \textbf{polymorphic behavior} is desired).             
    \end{itemize}

\texttt{std::unique\char`_ptr} may be constructed for an
\textbf{incomplete type} T, such as to facilitate the use as a handle 
in the pImpl idiom.

%% Shared Ptr
\subsection{\texttt{std::shared\char`_ptr}}
    \href{https://en.cppreference.com/w/cpp/memory/shared_ptr}{Cppreference documentation about \textbf{\texttt{std::shared\char`_ptr}}}.
    \newline
    This is a smart pointer that retains shared ownership of an object 
    through a pointer.
    \newline
    Several \texttt{shared\char`_ptr} objects may own the same object.
    \newline
    The object is destroyed and its memory deallocated when either of 
    the following happens:
    \begin{itemize}
        \item the last remaining \texttt{shared\char`_ptr} owning 
        the object is destroyed;
        \item the last remaining \texttt{shared\char`_ptr} owning 
        the object is assigned another pointer via \texttt{operator=} or 
        \texttt{reset()}.             
    \end{itemize}
    
    A \texttt{std::shared\char`_ptr} can share ownership of an object 
    while storing a pointer to another object.
    \newline
    A \texttt{std::shared\char`_ptr} may also own no objects, 
    in which case it is called empty (an empty 
    \texttt{std::shared\char`_ptr} may have a non-null stored pointer 
    if the aliasing constructor was used to create it).

%% Weak Ptr
  \subsection{\texttt{std::weak\char`_ptr}}
    \href{https://en.cppreference.com/w/cpp/memory/weak_ptr}{cppreference documentation about \textbf{\texttt{std::weak\char`_ptr}}}.
    \newline
    \texttt{std::weak\char`_ptr} is a smart pointer that holds a non-owning ("weak") reference 
    to an object that is managed by \texttt{std::shared\char`_ptr}. 
    It must be converted to \texttt{std::shared\char`_ptr} in order to access the referenced object. 
    \newline
    \texttt{std::weak\char`_ptr} models temporary ownership
